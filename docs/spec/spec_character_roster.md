## 2. 角色架构标准 (V3)

所有 8 个角色均遵循相同的 V3 架构：

### 2.1 文件结构

每个角色需要以下文件：

```
in_game/common/scripted_effects/gacha_{char}_effects.txt  # 核心逻辑
in_game/events/gacha_{char}_events.txt                    # 事件链
main_menu/common/static_modifiers/gacha_{char}_modifiers.txt  # 数值
```

### 2.2 核心Effect结构

| Effect名称 | 功能 |
|:---|:---|
| `gacha_create_{char}_effect` | 主入口：新建/升级角色 |
| `gacha_{char}_update_traits` | 命座对应的Trait变更 |
| `gacha_{char}_trigger_event` | 命座事件路由 |
| `gacha_{char}_apply_country_modifiers` | 国家Modifier应用 |

### 2.3 事件ID标准

| Event ID | 用途 | 触发时机 |
|:---|:---|:---|
| `.1` | 首抽传记 | 新角色创建时 |
| `.2` | 通用命座提升 | (备用) |
| `.4` | 满命成就 | C6达成时 |
| `.11` | C2觉醒事件 | C2达成时 |
| `.12` | C4超越事件 | C4达成时 |
| `.13` | C1初次突破 | C1达成时 |
| `.14` | C5事件 | C5达成时 |
| `.30+` | C3好感事件链 | C3达成时 |
